# 2. 基础命令

shell 和 bash 是包含和被包含的关系。

  • echo [字符串]，echo $变量名。
  • date 显示系统时间, timedatectl（time date controll） 用于设置系统时间。
  • 开关机，reboot、poweroff
  • 进程状态，ps。参数 a 表示显示所有进程，包括其他用户的进程；参数 u 表示用户和其他的详细信息；参数 x 表示没有控制终端的进程。

linux 中进程的状态分别为： 运行R，进程正在运行或运行队列中等待； 中断S，进程处于休眠中，等待信号或达成条件后脱离该状态，不可中断D，进程不响应系统异步信号，即便用 kill 命令也不能将其中断; 僵死Z，进程已经终止，但进程描述符依然存在, 直到父进程调用 wait4()系统函数后将进程释放; 停止T,进程收到停止信号后停止运行。

  • pstree 通过树形结构查看进程信息。
  • top 动态查看系统资源使用情况，类似 windows 中的任务管理器。
  • pidof 查询某个服务的进程的 pid。
  • kill、killall [pid]，：结束进程
  • 文件的查找：find，locate、whereis。

其中，find [范围] [参数] locate 使用 find 命令进行全盘搜索虽然更准确，但是效率有点低。如果仅仅是想找一些常见的 且又知道大概名称的文件，不如试试 locate 命令但是在使用前需要使用 updatedb 更新文件索引这个库文件的名字是/var/lib/mlocate/mlocate.db。
whereis 用于查找一个命令、二进制程序的位置如，whereis ls 用于查找 ls 命令在哪个位置。
其中 which 命令也可以用于查找命令、二进制程序的位置，which 命令是在 PATH 变量所指定的路径中，按照指定条件搜索命令所在的路径。也就是 说，如果我们既不关心同名文件（find 与 locate），也不关心命令所对应的源代码和帮助文件 （whereis），仅仅是想找到命令本身所在的路径。

# 文件编辑命令

  • cat 命令，查看文本内容
  • more、head、tail

head和tail同于查看文件的前面和后面的内容

  • tr （translate）用于对文本进行内容的替换，可以通过管道符，用在 cat 命令后。如cat lala.txt | tr lala wawa
  • stat 用于展示文件的状态，Linux 中文件有：最后访问时间（Access time）、最后修改时间（modify time）和文件属性最后修改时间（change time）。
  • grep 文本内容的搜索

参数可以有 -b 针对二进制文件；-c 只显示找到的行数；-I 忽略大小写；-n 显示行号；-v 反向选择，仅列出没有这个关键字的行号。在linux中，/etc/passwd 中保存着所有的用户信息，不允许登录的用户被保存在/sbin/nologin下。通过 grep 查找出当前不允许登录的用户信息： grep /sbin/nologin /etc/passwd

  • touch 用于创建一个空白文件、或设置文件的时间,用法为 touce [参数] 文件名

-a 用于修改文件的访问时间（access time）；-m，用于修改“修改时间（modify time）”

  • file 命令，linux 中文本、文件夹、目录、设备等都属于文件，通过 file 查看文件类型。
  • tar 压缩命令

-c 创建；-x 解压；-z 使用 gzip 进行解压缩；-C 指定解压到哪个目录；

# 管道符、重定向和环境变量

  • 通过 > 输出到文件中,man bash > lala.txt；通过 >> 追加内容到文件中；通过 &>> 将错误信息也输入到文件中如 ls A.txt >> lala.txt。文件 A 可能不存在，这时候会在终端输入错误信息，通过 &>> 将错误信息也输出到文件中ls A.txt &>> lala.txt
  • 管道命令符 | ，命令1 | 命令2 | 命令3，通过 | 将前一个的命令输出作为输入传递到后面去。
  • 文件通配符，* 任意字符、? 单个任意字符、[a-z] 小写字符、[A-Z]大小字母、[[:digital]] 任意数字.

如文件夹下有 sda1、sda2、sda3、sda4...等文件，如果需要一次查看它们，通过cat sda*。 也可以和创建文件命令组合，touch {a,b,c,d}.txt 一次性创建多个文件，通过 {} 名字之间通过逗号间隔。 echo file{1,2,3,4,5}.txt

  • 通过 XXX=XXX 的方式直接在终端中设置一个变量，这种变量具有作用域，一般是当前用户可以使用，如果切换到其他用户后不能使用，并且重新启动系统后，变量会丢失，除非设置到 .bashrc 或 .bash_profile 文件中。如果希望在终端中设置的变量，切换其他用户也可以使用需要 通过 export XXX 的方式导出变量，如果后续不需要这个变量通过 unset XXX 的方式取消。

# 接受用户参数

#!/bin/bash
echo "当前脚本名称为$0"
echo "总共有$#个参数，分别是$*"
echo "第 1 个参数为$1，第 5 个为$5。

$# 表示参数个数、$* 列出所有参数

#!/bin/bash
[ -d /etc/lala ] && echo "etc 下存在 lala 文件夹"
[ $USER = root ] || echo "|| 表示或"
[ !$USER = root ] && echo "当前用户不是 root"
[ $USER != root ] && echo "不是 root"

数字比较：
eq 等于 ne 不等于 gt 大于 lt 小于 ge 大于等于 le 小于等于

#!/bin/bash
INPUT_NUM=$1
INNER_NUM=5
[ $INPUT_NUM -gt $INNER_NUM ] && echo "gt 5"
[ $1 -ge $INPUT_NUM ] && echo "ge $1"

= 等于 != 不等于 -z 字符内容是否为空
